Задача:
Аналогично полям amount и profit добавить к сделке новые поля upsale и asapCharge
Оба поля integer от 0 до 999999 или NULL

Новые поля доступны для заполнения только если поля amount и profit больше 0.
Если передано пустое значение, то поля выставляются в значение NULL.

Сделайте удобно для пользователя (отключайте/скрывайте поле, если его нельзя менять) и безопасно для системы.

При изменении новых полей в истории диалога должна создаваться запись о том, что поле было изменено/поля были изменены.
Это делается кодом "$sms = (new Sms())..." по аналогии с amount и profit.

Внесите необходимые изменения в прикрепленные файлы, чтобы реализовать данную задачу.

---

Вы не можете протестировать работу кода. Пишите по аналогии с тем что уже есть.

Не используйте ИИ для решения данной задачи.

Идея состоит в том, чтобы вы ознакомились с реальным рабочим кодом, с которым предстоит работать, минимально разобрались в нем и смогли его изменить для реализации поставленной задачи. Это только небольшой кусок проекта, но позволит вам понять, с чем предстоить иметь дело.

Если что-то не понятно, уточните или используйте псевдокод.

В качестве результата пришлите измененные файлы.

---

К заданию прикреплен минимальный набор файлов, которые потребуют изменения.

DialogDeal.php - это entity объекта сделки

DealsController.php - это контроллер с кодом, который непосредственно изменяет сделки

dialog-form.html - это twig шаблон с формой диалога, которая всплывает при нажатию на сделку. 
Сам вызов и отправка результата формы осталась за кадром. Для вас важно, что:
- эта форма вызывается каждый раз, когда кто-то нажимает на создать или изменить сделку.
- сериализированные данные формы отсылается на twilio_sms_deal_create (и в случае редактирования и в случае создания сделки) ajax post запросом:
jQuery.post('{{ path('twilio_sms_deal_create') }}', jQuery('#deal-form').serializeArray(), function (data) {
  LOADER.hide();
  if (data) {
    if (data.error) {
      COMMON.error(data.error);
    } else if (data.success) {
      CRM.historyUpdate();
    } else if (data.task) {
      CRM.createReschedulingTask(dealId)
      CRM.historyUpdate();
    } else {
      COMMON.error('Unknown server response. Try refresh page');
    }
  } else {
    COMMON.error('Unknown error. Try refresh page');
  }
}).fail(function () {
  LOADER.hide();
  COMMON.error('Server error. Try again');
});
